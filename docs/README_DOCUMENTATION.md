# AI Studio Documentation Guide

Version: 1.0

Status: Permanent Documentation

---

# Purpose

This document provides an overview of the documentation system used by AI Studio and explains how the documentation is organized.

---

# Documentation Goals

The documentation is intended to:

- Explain the project.
- Document architectural decisions.
- Define development standards.
- Record project history.
- Assist contributors.
- Preserve long-term maintainability.

---

# Documentation Organization

The documentation is organized into the following categories:

| Category | Purpose |
|----------|---------|
| Core | Project fundamentals |
| Development | Development processes and standards |
| Technical | Architecture and implementation guidance |
| Quality | Testing and quality assurance |
| Reference | Supporting reference material |
| Templates | Reusable documentation templates |
| Historical | Archived documentation |

---

# Recommended Reading Order

New contributors should read:

1. `README.md`

5. `PROJECT_PHILOSOPHY.md`

# Stable Documents

These documents normally change only when project direction changes:

- PROJECT.md

- IMPLEMENTATION_PRINCIPLES.md

---

# Historical Records

Historical material is stored in:

```text
docs/old/
```

Historical documents are retained for reference only.

---

# Milestones

Implementation history is stored in:

```text
docs/milestones/
```

Each milestone should document:

- Objectives
- Work completed
- Testing
- Outstanding issues
- Completion status

---

# Documentation Rules

Documentation should:

- Remain accurate.
- Stay synchronized with implementation.
- Avoid duplication.
- Follow the project style guide.
- Maintain a single source of truth.
