# Glossary

Version: 1.0

Status: Permanent Documentation

---

# Purpose

This document defines the terminology used throughout AI Studio.

---

# A

## AI Studio

The application being developed.

A local-first AI video creation platform.

---

## API

Application Programming Interface.

Used for communication between application components.

---

# B

## Backend

The server-side application implemented using FastAPI.

Responsible for business logic and data management.

---

# C

## Component

A self-contained part of the application responsible for a specific task.

---

# F

## FastAPI

The Python web framework used by AI Studio.

---

## Filesystem Storage

The persistence mechanism used by the application.

All project data is stored locally.

---

# H

## HTMX

A frontend library used to update parts of web pages through server responses without building a full JavaScript application.

---

# J

## Jinja2

The server-side template engine used to generate HTML.

---

# L

## LM Studio

Local AI runtime providing an OpenAI-compatible API for language models.

---

# M

## Milestone

A defined stage of project development with its own objectives, implementation, testing, and completion criteria.

---

## Module

A source code file or collection of related functionality with a single responsibility.

---

# O

## OpenAI Compatible API

An API interface that allows different language models to be accessed using a common request format.

---

# P

## Project

A user workspace containing all generated data and assets.

---

## Prompt

Instructions sent to an AI model to generate output.

---

# R

## Route

A FastAPI endpoint that receives HTTP requests and delegates processing to services.

---

# S

## Service

A component responsible for implementing business logic.

Services should remain independent from presentation logic.

---

## Story

The initial narrative created during the content generation workflow.

---

## Scene

A smaller unit of a story used for image and video generation.

---

# T

## Template

A reusable document or file used as a starting point for new work.

---

# U

## User Interface

The web interface presented to the user.

Built using FastAPI, HTMX, and Jinja2.

---

# W

## Workspace

The local directory used to store application data, projects, generated assets, and intermediate files.

---

End of Glossary.
