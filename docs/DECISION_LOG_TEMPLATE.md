# Decision Log Template

Version: 1.0

Status: Template

---

# Purpose

This template provides a consistent format for documenting significant technical and architectural decisions.

Completed decision records should be summarized in:

`docs/DECISIONS.md`

---

# Decision Information

## Decision ID

```text
ADR-XXX
```

---

## Title

Provide a short descriptive title.

---

## Date

```text
YYYY-MM-DD
```

---

## Status

Choose one:

- Proposed
- Accepted
- Superseded
- Deprecated
- Rejected

---

# Context

Describe the problem or situation that required a decision.

Include:

- Background
- Constraints
- Requirements
- Existing implementation

---

# Options Considered

List all significant alternatives.

Example:

1. Option A
2. Option B
3. Option C

---

# Selected Option

Describe the chosen solution.

---

# Rationale

Explain why the selected option was chosen.

Include considerations such as:

- Simplicity
- Performance
- Maintainability
- Compatibility
- Development effort
- Long-term impact

---

# Consequences

Describe the expected effects.

Positive:

- Example benefit
- Example improvement

Negative:

- Trade-off
- Limitation

---

# Implementation Notes

Record implementation guidance if applicable.

---

# Related Documents

Examples:

- ARCHITECTURE.md
- PROJECT.md
- Current milestone

---

# Review

Record future review requirements if applicable.

---

End of Decision Log Template.
